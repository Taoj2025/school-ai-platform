### 3. 技术方案

#### 3.1 技术栈

- **前端**: Next.js 15 + TypeScript + Tailwind CSS
- **后端**: FastAPI + SQLAlchemy
- **数据库**: PostgreSQL

#### 3.2 文件结构

```
apps/web/app/dashboard/apple/{module}/
├── page.tsx              # 列表页
├── [id]/page.tsx         # 详情页
└── new/page.tsx          # 新增页

apps/api/app/modules/apple/{module}/
├── router.py             # API 路由
├── schemas.py             # Pydantic 模型
├── models.py              # ORM 模型
├── service.py             # 业务逻辑
└── repository.py          # 数据访问
```

#### 3.3 API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/apple/{module} | 列表 |
| POST | /api/v1/apple/{module} | 创建 |

**请求示例**:
```json
{
  "name": "示例",
  "value": 100
}
```

**响应示例**:
```json
{
  "data": {
    "id": "001",
    "name": "示例",
    "value": 100
  }
}
```

### 6. AI 辅助开发提示

#### 6.1 参考文件

- 参考页面: `apps/web/app/dashboard/apple/{module}/page.tsx`
- 参考组件: `components/modules/apple/{Component}.tsx`
- 参考 API: `apps/api/app/modules/apple/{module}/router.py`

#### 6.2 代码规范

**前端**:
- 使用 CSS 变量替代硬编码颜色
- 使用 `inputStyle` 统一表单样式
- 使用 `ThemeProvider` 管理主题

**后端**:
- 使用 Pydantic 进行数据验证
- 使用 SQLAlchemy 进行数据库操作
- 遵循 RESTful API 设计规范

#### 6.3 常见问题

**Q: 如何处理主题切换？**
A: 使用 CSS 变量 `var(--text)`, `var(--bg)`, `var(--panel)` 等

**Q: 如何处理加载状态？**
A: 使用 `useState` 管理 loading 状态
